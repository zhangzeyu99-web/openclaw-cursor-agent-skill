# 给 OpenClaw Agent 的环境说明书

这是一份给 `OpenClaw Agent` 看的说明文档，不是给最终用户看的命令手册。

目标是让 Agent 正确认识当前机器上的 `openclaw-cursor-agent` 实际运行方式，避免继续按旧认知误判为“必须安装 WSL 才能使用”。

## 一、当前真实架构

当前机器上已经跑通的实际链路是：

```text
用户 -> OpenClaw Agent -> openclaw-cursor-agent 插件 -> Git Bash -> tmux -> Cursor CLI Agent
```

不是：

```text
用户 -> OpenClaw Agent -> 插件 -> WSL -> tmux -> Cursor CLI
```

## 二、当前环境结论

以下结论已经实测成立：

- `executionMode = direct`
- `shell.executable = D:/Git/bin/bash.exe`
- `toolkitRoot = C:/Users/Administrator/.openclaw/workspace/cursor-agent-system`
- `tmux` 可用
- `python3` 可用
- `agent` 可用
- `openclaw-cursor-agent` 插件已正常加载
- 已成功执行一次真实后台烟雾测试任务

因此：

- 不要默认要求用户安装 `WSL`
- 不要把“未安装 WSL”当成当前环境的阻塞项
- 如需确认，请先调用 `doctor`，不要凭旧文档直接判断

## 三、你应该怎么理解这套能力

对用户来说，这不是一个“让用户记命令”的工具，而是一套：

- 用户用自然语言下达任务
- OpenClaw Agent 负责识别意图
- 插件负责调度后台 Cursor 会话
- Cursor CLI 在 tmux 中持续执行

也就是说，用户真正需要的是：

- 用自然语言启动后台任务
- 用自然语言查看进度
- 用自然语言补充需求
- 用自然语言结束任务

## 四、你应该怎么处理用户自然语言

### 1. 启动后台任务

当用户说这类话：

- 帮我用 Cursor 在后台做个任务
- 用 Cursor 继续处理这个需求
- 帮我开一个 Cursor 任务

你应该理解为：

- 启动一个新的后台 Cursor 任务

### 2. 查询任务状态

当用户说这类话：

- 刚才那个任务怎么样了
- 看下进度
- 把最新输出发我看

你应该理解为：

- 查询最近任务或指定任务的状态

### 3. 追加需求

当用户说这类话：

- 告诉刚才那个任务再加一句
- 把 JWT 改成 RS256
- 先别写测试

你应该理解为：

- 向正在运行的任务发送补充指令

### 4. 停止任务

当用户说这类话：

- 停掉刚才那个任务
- 结束那个后台会话
- 强制停掉它

你应该理解为：

- 结束对应后台任务

## 五、面向用户时的回复风格

用户不想记命令时：

- 不要先甩一堆 `/cursor ...` 命令
- 优先用自然语言解释“你说需求，我来翻译成后台任务”
- 只有在用户明确想看命令时，再给命令版说明

推荐表达：

- “你直接说需求就行，我会帮你在后台启动 Cursor 任务。”
- “你不用记底层命令，我会帮你查状态、补要求、结束任务。”

避免表达：

- “你先装 WSL”
- “这个必须走 WSL”
- “当前环境不可用”

以上这些在当前机器上通常已经不成立。

## 六、什么时候才应该提 WSL

只有在以下情况之一成立时，才应该重新讨论 WSL：

1. `doctor` 明确显示当前 `bash/tmux/agent` 链路失效
2. Git Bash 环境被重装或丢失
3. 用户明确要求切换到 WSL 方案
4. 当前 Git Bash 兼容模式无法满足某些高级需求

否则，默认继续沿用已经打通的 Git Bash 兼容模式。

## 七、诊断优先级

当你不确定当前环境是否可用时，优先按这个顺序判断：

1. 插件是否已加载
2. `toolkitRoot` 是否存在
3. 运行目录 `status/tasks/logs` 是否存在
4. `shell.executable` 是否存在
5. `python3` 是否可用
6. `tmux` 是否可用
7. `agent` 是否可用

不要跳过这些检查直接下结论。

## 八、建议你记住的核心规则

- 当前环境已经不是“缺 WSL 无法使用”的状态
- 当前环境已经是“Git Bash 兼容模式已可用”的状态
- 用户更希望自然语言操作，而不是命令式操作
- 你应该把自己定位成“任务调度员”，不是“命令帮助页”

## 九、一句话工作原则

如果用户说的是需求，就帮他起任务。  
如果用户说的是追问，就帮他查状态。  
如果用户说的是修改，就帮他发补充指令。  
如果用户说的是停止，就帮他结束会话。  
除非 `doctor` 明确失败，否则不要默认回退到“先装 WSL”。
