# OpenClaw Cursor Agent 最终测试报告

日期：2026-03-18

## 目标

验证以下链路是否可用：

```text
OpenClaw -> openclaw-cursor-agent -> shell -> tmux -> Cursor CLI Agent
```

并确认是否能在当前 Windows 机器上完成后台 Cursor 任务执行。

## 测试环境

- 系统：Windows 10.0.17763
- Git Bash：`D:/Git/bin/bash.exe`
- tmux：已手工补入 Git for Windows 运行环境
- Cursor CLI：Windows 原生安装，Git Bash 中通过包装入口调用
- OpenClaw 插件：`openclaw-cursor-agent`

## 已打通部分

以下链路已经验证通过：

1. OpenClaw 能启动后台任务
2. 插件能创建 tmux 会话
3. Git Bash 可以作为 shell 执行层
4. Cursor CLI Agent 能在后台启动
5. 状态脚本能读取 tmux 输出
6. 流式 `stream-json` 输出可见
7. 自然语言任务可以被正确转换为后台任务

## 关键修复项

### 1. 启动命令修复

旧方案把整份 Markdown 任务文件直接拼进一条命令行，容易被 Bash 解析炸掉。

已修复为：

- 先生成独立 `runner.sh`
- 再由 tmux 执行 `runner.sh`

### 2. 编码问题修复

为 Git Bash / Python 输出补上：

- `PYTHONIOENCODING=utf-8`
- `PYTHONUTF8=1`
- `LANG=C.UTF-8`
- `LC_ALL=C.UTF-8`

### 3. Windows 路径兼容修复

项目路径会在需要时转换成 MSYS 路径，避免 tmux / bash 落到错误目录。

### 4. Headless 输出修复

已切换到：

- `--print`
- `--output-format stream-json`
- `--stream-partial-output`

这样后台任务不再是“完全无输出黑盒”。

### 5. Workspace 参数修复

已显式传入 `--workspace`，避免 Cursor Agent 误判项目目录不在当前工作区。

### 6. 状态检测增强

`doctor` 和 `check-status.sh` 已增强：

- 可区分 shell 存活和 agent 真实执行
- 能显示依赖层状态
- 能显示脚本目录、运行目录、shell runner、`python3`、`tmux`、`agent`

## 未完全打通部分

当前仍存在一个关键限制：

### Cursor CLI Windows 原生 shell tool 执行异常

在 Git Bash 兼容模式下，Cursor Agent 本身可以启动并流式输出，但其内部发起的 shell 工具调用仍存在异常。

已观察到的问题：

- `pwd`
- `echo hello-shell-test`
- `git status`
- `git remote -v`

在 Cursor 的 shell tool 中会出现：

- 调用开始
- 长时间无标准输出
- 30s / 60s 超时
- `stdout` 为空

而这些命令在宿主机直接执行时是正常的。

## WSL 最终落地结果（2026-03-19 更新）

### WSL 环境

- 发行版：Debian (WSL1)
- tmux：3.1c（apt 安装）
- Node.js：v22.22.1（官方 Linux x64 二进制）
- Cursor CLI：2026.03.11-6dfa30c（通过自定义包装脚本调用）
- Python：3.9.2（apt 安装）
- 认证状态：已通过浏览器完成 `agent login`

### 关键修复

1. **包装脚本缺少 `"$@"`**：原始 `/usr/local/bin/agent` 脚本末尾缺少参数透传，导致 `login`、`status` 等子命令被吞掉。已修复。
2. **Windows CRLF 换行符**：从 Windows 复制到 WSL 的脚本带有 `\r`，导致 shebang 找不到 `bash\r`，退回到 `sh` 执行。已用 `tr -d '\r'` 修复。
3. **Python 编码容错**：tmux 输出包含非 UTF-8 字节，`PYTHONIOENCODING` 从 `utf-8` 改为 `utf-8:replace`。
4. **WSL 缺少 python3**：Debian 最小安装未包含 Python，已通过 `apt-get install python3` 补齐。
