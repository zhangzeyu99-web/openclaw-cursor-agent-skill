# Cursor 后台任务自然语言使用说明

这套能力可以理解成：

- `OpenClaw` 负责听你说话、理解需求、调度任务
- `Cursor` 负责真正执行编码任务
- `tmux` 只是后台会话机制，你不用关心

你以后不需要记很多命令，尽量直接用自然语言说需求就行。

## 一句话理解

你把 OpenClaw 当成一个项目经理，对它说：

- 开一个 Cursor 后台任务
- 看看刚才那个任务做到哪了
- 给刚才那个任务补充一句要求
- 把刚才那个任务停掉

## 你可以直接这么说

### 1. 开一个 Cursor 后台任务

直接说：

- 帮我用 Cursor 在后台处理一个任务：给 openclaw 项目补一个 README，不要动其他文件。
- 帮我开一个 Cursor 任务，在 `D:/project/openclaw` 里实现 JWT 登录接口。
- 用 Cursor 后台继续做这个需求，项目路径是 `D:/project/openclaw`。

## 2. 查看任务进度

直接说：

- 看看刚才那个 Cursor 任务做到哪了。
- 帮我查一下后台 Cursor 任务进度。
- 把刚才那个任务的最新输出给我看一下。

## 3. 给任务补充要求

直接说：

- 告诉刚才那个 Cursor 任务：JWT 改成 RS256。
- 给刚才那个任务补一句，不要修改无关文件。
- 让它先别写测试，先把接口实现完。

## 4. 暂停、恢复、结束

直接说：

- 先暂停刚才那个 Cursor 任务。
- 恢复刚才那个 Cursor 任务。
- 把刚才那个 Cursor 任务停掉。
- 强制结束刚才那个后台 Cursor 会话。

## 最推荐的使用方式

你以后就按这个节奏来：

1. 先直接说需求
2. 让 OpenClaw 帮你起 Cursor 后台任务
3. 中途继续补充要求
4. 随时让它汇报进度
5. 做完后让它停掉

## 一个完整例子

你可以直接这么说：

```text
帮我用 Cursor 在后台做一个任务：给这个项目补 README，要求中文，不要修改其他文件。项目路径是 D:/project/openclaw。
```

过一会你再说：

```text
看一下刚才那个 Cursor 任务进度。
```

如果你想临时改要求：

```text
告诉刚才那个任务：README 里加一个“快速开始”章节。
```

最后结束：

```text
把刚才那个 Cursor 任务停掉。
```

## 你不用关心的东西

这些都已经在底层接好了，你不用自己处理：

- `tmux`
- `agent`
- `Git Bash`
- 后台会话
- 状态文件
- 任务日志

## 结论

以后最简单的理解就是：

不要把它当成“命令行工具”，  
要把它当成“你在指挥一个会后台持续工作的 Cursor 工程师”。
